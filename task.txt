
Описание
Создайте простой RESTful API с использованием FastAPI для приложения социальной сети.
Функциональные требования:
Должна быть некоторая форма аутентификации и регистрации (JWT, Oauth, Oauth 2.0 и т. д.).
В качестве пользователя я должен иметь возможность зарегистрироваться и войти в систему.
В качестве пользователя я должен иметь возможность создавать, редактировать, удалять и просматривать записи.
В качестве пользователя я могу ставить лайки или дизлайки записям других пользователей, но не своим собственным.
API должен иметь документацию пользовательского интерфейса (Swagger/ReDoc).

Раздел "бонус" (необязательно):
Используйте https://clearbit.com/platform/enrichment для получения дополнительных данных о пользователе при регистрации.
Используйте emailhunter.co для проверки существования электронной почты при регистрации.
Используйте базу данных в памяти для хранения лайков и дизлайков записей (в качестве кэша, который обновляется при добавлении новых лайков и дизлайков).

Требования к технологиям
Задачи должны быть выполнены:
С использованием FastAPI 0.50.0+
С использованием любой СУБД (Sqlite, PostgreSQL, MySQL)
Загружены на GitHub
Требования
При реализации вашего решения, убедитесь, что код:
Хорошо структурирован
Содержит инструкции (лучше всего поместить их в readme.md) о том, как развернуть и протестировать его
Чистый
Программа, которую вы реализуете, должна быть полноценным продуктом, то есть должна быть легко устанавливаемой, предусматривать обработку нестандартных ситуаций, быть устойчивой к неправильным действиям пользователя и т. д.
